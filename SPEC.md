## Overview

Squire is a file-based toolkit that changes how AI coding agents behave. It ships as a collection of markdown files and shell scripts that you drop into your project or global config directory. The core artifact is `squire.md`, a behavioral ruleset that addresses the 9 most common failure patterns observed across 5 months of intensive AI-augmented development. Around that core sit 63 slash commands, 328 skills, 34 custom agents, an 11-stage build pipeline, 7 behavioral patterns, 6 thinking frameworks, the Triad documentation system, and workspace generators for file-based project organization.

## Installation

Three install paths, all equivalent:

1. **Curl single file** -- `curl -fsSL` the raw `squire.md` into project root or `~/.claude/CLAUDE.md`
2. **Full installer** -- `./install.sh` with flags for `--dry-run`, `--commands-only`, `--uninstall`
3. **Cherry-pick** -- Copy individual directories (`commands/`, `skills/`, `prompts/`) as needed

## Design System

All HTML artifacts generated by Squire skills follow the Factory-Inspired design language: near-black (#020202), near-white (#eeeeee), orange (#ef6f2e), amber (#f59e0b), teal (#4ecdc4). Geist + Geist Mono. No shadows, gradients, or glow effects.

## Multi-Model Orchestration

The Director/Builder pattern supports two AI models working together. The Director (reasoning model) plans, reviews, integrates, and ships. The Builder (code generation model) executes scoped implementation tasks. Failure threshold: 2 Builder failures on the same task triggers Director takeover.

## Dependencies

- Bash 3.2+ (for workspace generators)
- Node.js (for projects using the pipeline)
- Claude Code CLI (primary target surface)
- No runtime dependencies, no npm packages, no build steps
